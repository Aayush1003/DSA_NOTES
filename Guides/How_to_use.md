# How to use this repository

This short guide explains how the repo is organized and how to add or use notes and problems effectively.

1. Browse folders by topic. Each top-level folder groups related notes and patterns.
2. When learning a topic:
   - Read the overview file (if present).
   - Implement the canonical data structure or algorithm from the note.
   - Solve multiple practice problems and record one note per problem using the template.
3. Naming conventions:
   - Topic folders: `2_Data_Structures/`, `3_Algorithms/` (already present)
   - Problem notes: use a short slug and include difficulty and tags in the header.
4. Adding new content:
   - Create a markdown file in the appropriate folder.
   - If adding many problems or a new subtopic, open a PR describing the change.
5. Problem tracking:
   - Use `Guides/Problem_Template.md` to record your approach, tests, and lessons.

If you'd like automation (progress tracker, per-topic checklists, or curated problem sets), I can scaffold that next.
