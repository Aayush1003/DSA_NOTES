# Study Roadmap

This document suggests a focused, practical path through the notes in this repo. It maps topics to patterns, gives a sample weekly plan, and provides study habits that worked for many engineers.

## Recommended study order (high level)
1. Basics & Complexity (Big-O, space/time tradeoffs)
2. Arrays & Strings: sliding window, two pointers, cyclic sort
3. Linked Lists, Stacks & Queues
4. Sorting & Searching: binary search variants, heaps (Top-K)
5. Trees: traversal, BFS/DFS, binary trees, BST
6. Graphs: BFS, DFS, Dijkstra, topological sort
7. Two Pointers & Sliding Window patterns (window sums, variable windows)
8. Dynamic Programming: knapsack patterns, LIS, palindromes, interval DP
9. Backtracking & subsets
10. Review + mixed problem sets and timed mocks

## Key patterns (quick mapping)
- Array & String: Sliding Window, Two Pointers, Fast & Slow Pointers, Merge Intervals, Cyclic Sort, Monotonic Stack
- Search & Sort: Modified Binary Search, Top K (Heaps), K-way Merge
- Tree & Graph: BFS, DFS, Topological Sort
- Backtracking & Subsets: Subsets / Backtracking, Bitwise XOR tricks
- Dynamic Programming: 0/1 Knapsack, Unbounded Knapsack, LIS, Palindrome DP, Matrix/Interval DP

## Example 8-week plan
- Week 1: Big-O + Arrays (basics + implementation)
- Week 2: Two pointers, Sliding Window, and practice problems
- Week 3: Linked Lists + Stacks/Queues
- Week 4: Sorting algorithms + Binary Search variants
- Week 5: Trees (traversal, common problems)
- Week 6: Graphs (BFS/DFS, shortest paths)
- Week 7: Dynamic Programming (start with knapsack / LIS)
- Week 8: Backtracking, review, timed mock interviews

## Daily routine (sample)
- 20–30m: read/implement the core idea and code it
- 30–60m: solve 2–4 problems (one easy, one medium, optionally one hard)
- 10–15m: write a short problem note (link to `Guides/Problem_Template.md`)

## Practice targets
- Aim for 50–150 focused problems across topics for solid coverage.
- For each pattern: implement one canonical solution and solve 10–20 related problems.

## Resources
- LeetCode (problem tags), GeeksforGeeks, HackerRank
- Textbooks: CLRS (selected chapters), Elements of Programming Interviews (EPI)
